# ตรวจเส้นทาง/เมนู (Route/Menu Audit)

ใช้คู่กับหน้า `/admin/health` และ `bun run e2e:smoke` เพื่อตรวจว่าเมนูหลักในเว็บมีผลลัพธ์จริง ไม่ใช่ปุ่มหลอกก่อน deploy

| พื้นที่ | Route | ปุ่ม/เมนู | ผลลัพธ์จริง | เงื่อนไขปิดปุ่ม/Guard | สถานะว่าง |
| --- | --- | --- | --- | --- | --- |
| สำรวจ / หน้าแรก | `/` | ค้นหา, หมวดหมู่, การ์ดตัวละคร, เล่นต่อ, สร้างตัวละคร, เมนูล่างมือถือ | เปิดหน้าตัวละคร, เปิดแชทเดิม, ไปหน้าสร้างตัวละครได้จริง และมือถือมีเมนูไปหน้าหลักครบ | ถูกปิดเฉพาะตอนโหลดข้อมูล | มีข้อความโหลดผิดพลาดและการ์ดตัวอย่างกันหน้าว่าง |
| หน้าโปรไฟล์ตัวละคร | `/characters/:characterId` | สัญญาความสัมพันธ์, เริ่มแชท, คัดลอกลิงก์, รายงาน | ส่ง `relationship_seed` ไปหน้าแชทและเปิดหน้าต่างรายงาน | รายงานถูกปิดระหว่างส่ง | มีข้อความบอกเมื่อโหลดตัวละครไม่ได้ |
| ห้องแชท | `/chat`, `/chat/:chatId` | ช่องพิมพ์, แถบเครื่องมือ, ฉากที่รออยู่, โหมดอ่าน, รายงาน, โปรไฟล์ | ส่งข้อความ, เปิดพรอมป์ฉาก, โหมดอ่านเปลี่ยนพื้นที่อ่านข้อความจริง, รายงาน, ทางไปกระเป๋า/โปรไฟล์ | ปุ่มส่งถูกปิดเมื่อข้อความว่าง, กำลังสตรีม, หรือโทเคนไม่พอ | ห้องใหม่มีบทนำ/คำทักทาย |

| สร้างตัวละคร (Creator Studio) | `/create` | AI สร้างรูป+เนื้อหา, อัปโหลด, ลิงก์รูป, ตัวช่วยเช็กแท็ก, ตัวลองบท, เผยแพร่ | สร้างร่างได้จริง มีภาพตัวอย่างสำรองถ้ายังไม่มีผู้ให้บริการรูปภาพ | เผยแพร่ถูกปิดเมื่อข้อมูลหลักไม่ครบ, แท็กชนระดับอันตราย, หรือกำลังบันทึก | แผงความพร้อมบอกช่องที่ยังขาด |
| กล่องแชท | `/chats` | ค้นหา, ตัวกรอง, เปิดแชท, ป้ายฉากที่รออยู่, เมนูสามจุด, เลือกหลายแชท, กู้คืนแชท | เปิดกลับไป `/chat/:chatId` ได้ เมนูแก้ชื่อ/ปักหมุด/จัดเก็บ/ลบมีผลจริง จัดเก็บ/กู้คืน/ลบหลายรายการถูก smoke บน desktop/mobile แล้ว | ถูกปิดเฉพาะตอนกำลังบันทึกหรือยังไม่ได้เลือกแชท | แต่ละตัวกรองมีสถานะว่างของตัวเอง เช่น ยังไม่มีแชทปักหมุด/จัดเก็บ |
| กล่องอีเวนต์ | `/events` | ฉากที่รออยู่แบบจัดกลุ่มตามฉาก พร้อมแชทย่อยแต่ละห้อง | กดแชทย่อยในกลุ่มอีเวนต์แล้วเปิดกลับเข้าห้องที่มีฉากรออยู่ | อีเวนต์หมดอายุไม่แสดงคำสั่งหลัก | บอกว่ายังไม่มีฉากสำคัญ |
| โปรไฟล์ / ตัวตนผู้เล่น | `/profile` | ตัวตนผู้เล่น, ตั้งค่าเนื้อหา, ตัวละครที่สร้าง/ชื่นชอบ | เซฟตัวตนผู้เล่นลงบัญชีและร่างในเครื่อง พร้อมใช้เป็นบริบทในพรอมป์แชท ส่วนโหมดเนื้อหาบันทึกผ่าน backend | ถูกปิดตอนกำลังบันทึก | บอกให้เริ่มสร้าง/สำรวจเมื่อยังไม่มีข้อมูล |
| กระเป๋าโทเคน | `/wallet` | รีเฟรช, คีย์แอดมิน, เพิ่ม/หักโทเคน, ประวัติใช้งาน, ประวัติธุรกรรม | โหลดโทเคน/ธุรกรรมจาก backend และมี QA seed data | เพิ่ม/หักถูกปิดถ้าไม่มีคีย์แอดมิน, ไม่มีสรุปผู้ใช้, จำนวนไม่ถูกต้อง, หรือกำลังส่ง | บอกว่ายังไม่มีการใช้งาน/ธุรกรรม |
| ดูแลรายงาน (Moderation) | `/moderation` | คีย์แอดมิน, ตัวกรอง, ค้นหา, เปิดต้นทาง, คำสั่งจัดการ, audit log | ตรวจคิวรายงานและ audit log ได้ | guard ด้วย `ADMIN_API_KEY` | บอกวิธีทดสอบ flow รายงาน |
| ตรวจระบบผู้ดูแล | `/admin/health` | รีเฟรชสถานะ, สรุป blocker production, checklist deploy, ตรวจเส้นทาง/เมนู, ลิงก์ตรวจพรอมป์, ลิงก์ทดสอบคุณภาพ | เห็น DB, AI, Supabase, storage, CORS, production blocker count, route audit, ขั้นต่อไปของแต่ละ blocker และคำสั่ง final gate ในหน้าเดียว พร้อมไปตัวตรวจพรอมป์/ชุดทดสอบอัตโนมัติได้ | ไม่มีปุ่มที่ถูกปิดถาวร รีเฟรชถูกปิดได้เฉพาะตอนเรียกข้อมูล | backend ล่มจะแสดงสิ่งที่ต้องแก้ |
| ตัวตรวจพรอมป์ | `/admin/prompt-inspector` | คีย์แอดมิน, เลือกตัวละคร, ข้อความปัจจุบัน, ข้อความก่อนหน้า, โน้ต runtime, persona ที่แทนค่า, ตรวจพรอมป์, คัดลอกพรอมป์ที่ปิดข้อมูลลับ | เรียก admin API เพื่อตรวจภาพรวมพรอมป์แบบปิดข้อมูลลับ, งบโทเคนรายส่วน, คลังความรู้ที่ดึงมาใช้, คำเตือน และส่วนต่างของพรอมป์ โดยไม่ยิงโมเดลจริง | guard ด้วย `ADMIN_API_KEY`; ปุ่มตรวจถูกปิดเมื่อยังไม่พร้อมหรือกำลังตรวจ | แสดงสถานะยังไม่ได้ตรวจพรอมป์และบอกให้เลือกตัวละคร/ข้อความก่อนตรวจ |
| ชุดทดสอบอัตโนมัติ | `/admin/evals` | คีย์แอดมิน, ปุ่มรันชุดทดสอบ, แผงพับชุดทดสอบ, ผลเช็ก, สรุปจุดไม่ผ่าน | เรียก admin API เพื่อรันชุด roleplay แบบผลซ้ำได้ ตรวจกฎคุมพรอมป์, คลังความรู้, ความต่อเนื่องของความสัมพันธ์/ฉาก และงบโทเคน โดยไม่ยิงโมเดลจริง | guard ด้วย `ADMIN_API_KEY`; ปุ่มรันชุดทดสอบถูกปิดเมื่อไม่มี key หรือกำลังรัน | แสดงสถานะยังไม่ได้รันชุดทดสอบ และอธิบายว่าจะเห็นชุดทดสอบ/ผลเช็ก/จุดไม่ผ่านหลังรัน |
| สเตจจิงจริง | โดเมน staging จริง | Supabase จริง, backend บน Render/Railway, frontend domain, CORS จริง, e2e smoke | ต้องผ่าน staging ก่อน production | ต้องใช้บัญชีและ domain จริง | ใช้ `STAGING_RUNBOOK.md` |

Automated route smoke ล่าสุดเช็คทุก route หลักบน desktop/mobile, ดัก browser console/page error, และเช็ค horizontal overflow เพื่อจับปัญหาปุ่มล้นหรือหน้า admin ยิง API ทั้งที่ยังไม่มีสิทธิ์
เมนูแชทถูกเช็คทั้งสองพื้นผิวคือ sidebar ในห้องแชทและหน้า `/chats` รวมถึง pin/unpin, rename, archive, restore, delete, selection mode และ bulk archive/restore/delete

คำสั่ง smoke:

```bash
bun run route-menu:audit
bun run qa:seed
bun run e2e:smoke
bun run qa:full
```

Automated guard ล่าสุด:

- `scripts/route-menu-doc-check.ts` ตรวจให้ `ROUTE_MENU_AUDIT.md` มีทุก route/menu ที่แสดงใน `/admin/health`, route จริงใน `App.tsx`, nav path, และ route preload ครบตรงกัน ถ้าตารางหรือเมนู drift ออกจาก source จริงจะ fail ทันที
- `scripts/frontend-static-audit.ts` ตรวจปุ่ม `<button>` ที่ไม่มี `type`, ปุ่มไอคอนที่ไม่มี `aria-label`/`title`, ลิงก์ placeholder `href="#"`/`to="#"`, handler ว่าง, ข้อความ placeholder ที่ไม่ควรหลุดขึ้น production, และ text encoding ที่เพี้ยนเป็น mojibake
- `scripts/frontend-route-audit.ts` ตรวจลิงก์/เมนู static ที่ชี้ไป route จริงใน `App.tsx` เพื่อกันเมนูที่กดแล้วตัน
- `scripts/check-frontend-bundles.ts` ล็อก bundle budget โดยเฉพาะ main bundle และหน้า Chat ที่ถูก lazy load
