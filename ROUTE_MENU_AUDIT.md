Smoke เส้นทางอัตโนมัติล่าสุดเช็คทุก route หลักบนจอเดสก์ท็อปและมือถือ, ดักข้อผิดพลาดในคอนโซล/หน้าเบราว์เซอร์, และเช็คการล้นแนวนอนเพื่อจับปัญหาปุ่มล้นหรือหน้า admin ยิง API ทั้งที่ยังไม่มีสิทธิ์
Smoke ยังล็อกหน้าประสบการณ์เต็มจอของ `/` และ `/chat*` ว่าต้องไม่ถูก shell กลางครอบซ้ำ, ต้องไม่มี `app-mobile-nav` ของ shell กลาง, และต้องไม่แสดงข้อความ `โหมดสว่างยังไม่รองรับ`
เมนูแชทถูกเช็คทั้งสองพื้นผิวคือ sidebar ในห้องแชทและหน้า `/chats` รวมถึง pin/unpin, rename, archive, restore, delete, selection mode และ bulk archive/restore/delete

คำสั่ง smoke:

```bash
bun run frontend:static:audit
bun run frontend:route:audit
bun run route-menu:audit
bun run qa:seed
bun run e2e:smoke
bun run qa:full
```

Automated guard ล่าสุด:

- `scripts/route-menu-doc-check.ts` ตรวจให้ `ROUTE_MENU_AUDIT.md` มีทุก route/menu ที่แสดงใน `/admin/health`, route จริงใน `App.tsx`, nav path, และ route preload ครบตรงกัน ถ้าตารางหรือเมนู drift ออกจาก source จริงจะ fail ทันที
- `tests/e2e/maprang-smoke.spec.ts` ตรวจหน้าประสบการณ์เต็มจอของ `/` และ `/chat*` ว่าไม่ถูก shell กลางครอบซ้ำ, ไม่มี `app-mobile-nav` ของ shell กลาง, และไม่มีข้อความ `โหมดสว่างยังไม่รองรับ`
- `scripts/frontend-static-audit.ts` ตรวจปุ่ม `<button>` ที่ไม่มี `type`, ปุ่มไอคอนที่ไม่มี `aria-label`/`title`, ลิงก์ตัวอย่าง `href="#"`/`to="#"`, ตัวจัดการคลิกว่าง, ข้อความตัวอย่างที่ไม่ควรหลุดขึ้นก่อนปล่อยใช้งานจริง, และการเข้ารหัสข้อความที่เพี้ยน
- `scripts/frontend-route-audit.ts` ตรวจลิงก์/เมนู static ที่ชี้ไป route จริงใน `App.tsx` เพื่อกันเมนูที่กดแล้วตัน
- `scripts/check-frontend-bundles.ts` ล็อก bundle budget โดยเฉพาะ main bundle และหน้า Chat ที่ถูก lazy load
